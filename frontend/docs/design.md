# 할리갈리(HaliGali) 프론트엔드 디자인 계획 (개정판)

본 문서는 마우스 대기 방지 로직(랜덤 종 위치)이 포함된 할리갈리 전용 게임 화면의 UI/UX 설계를 정의합니다.

## 1. UI 레이아웃 설계 (4인 최적화)

화면은 플레이어들을 상하좌우로 배치하고, 중앙의 가변적인 영역에서 게임이 진행되는 구조입니다.

- **플레이어 배치 (상하좌우)**:
  - **Bottom (나)**: 화면 하단 중앙. 내 덱과 오픈된 카드가 가장 크게 보임.
  - **Top**: 화면 상단 중앙. 상대방의 정보 표시.
  - **Left / Right**: 화면 좌우측 중앙. 상대방의 정보 표시.
  - **구성**: 닉네임, 남은 카드 수(덱), 현재 바닥에 오픈된 카드.
- **중앙 게임 구역 (Game Arena)**:
  - 4명의 플레이어가 뒤집은 카드들이 모이는 중앙 공간.
  - **랜덤 종 (Dynamic Bell)**:
    - 종은 고정된 위치가 아니라, 중앙 구역 내의 무작위 좌표에 생성됨.
    - 카드가 뒤집힐 때마다 위치를 변경하여 마우스 캠핑을 방지함.
- **제거된 요소**: 퀴즈용 타이머, 실시간 랭킹 패널 (게임에 집중하기 위해 제거).

## 2. 핵심 게임 로직 및 UX

1. **마우스 대기 방지 (Anti-Camping Bell)**:
   - 종 버튼은 `absolute` 포지션을 가지며, 중앙 아레나의 특정 범위(예: 300x300px) 내에서 랜덤한 `top`, `left` 값을 가짐.
   - 종이 나타날 때 가벼운 애니메이션(나타나기 효과)을 주어 시각적 인지 유도.
2. **턴 표시 및 강조**:
   - 현재 턴인 플레이어의 영역(닉네임 및 카드)을 밝은 테두리로 강조.
   - 내 턴일 때만 화면 하단에 "카드 뒤집기" 버튼 활성화.
3. **카드 애니메이션**:
   - 플레이어 덱에서 중앙 오픈 구역으로 카드가 날아가는 `Framer Motion` 또는 `CSS Transition` 적용.
   - 종 울리기 성공 시 바닥의 모든 카드가 승자 쪽으로 빨려 들어가는 연출.

## 3. 컴포넌트 구조

- `HaliGaliPage.tsx`: 전체 레이아웃 및 웹소켓 데이터 관리.
- `Arena.tsx`: 중앙의 오픈 카드들과 랜덤하게 움직이는 **Bell**을 관리.
- `PlayerZone.tsx`: 각 위치(Top, Bottom, Left, Right)에 따른 플레이어 정보 렌더링.
- `HaliGaliCard.tsx`: 과일 테마 카드 UI.

## 4. 데이터 연동 스펙

- **`HALIGALI_ACTION` 메시지 수신**:
  - `status` 데이터를 기반으로 4개 영역의 카드 정보를 실시간 동기화.
  - `result === 'CORRECT'` 인 경우 종의 위치를 즉시 랜덤하게 재설정.
- **API 호출**:
  - 종 클릭 시: `POST /game/rooms/{roomId}/action` (type: `PRESS_BELL`).
  - 카드 클릭/버튼 클릭 시: `POST /game/rooms/{roomId}/action` (type: `FLIP_CARD`).
