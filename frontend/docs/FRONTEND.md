# 프론트엔드 작업 가이드라인 (FRONTEND.md)

## 기술 스택
- **프레임워크:** React (Vite)
- **언어:** TypeScript
- **상태 관리:** (현재 사용 중인 방식 준수)
- **스타일링:** Vanilla CSS (기존 프로젝트 스타일 유지)

## 작업 지침
1. **컴포넌트 개발:** `src/components` 및 `src/pages` 구조를 유지하며 재사용성을 고려합니다.
2. **타입 안전성:** TypeScript 타입을 명확히 정의하고 `src/types`에 관리합니다.
3. **상태 관리 및 Hooks:** 비즈니스 로직은 가능한 `src/hooks`로 분리하여 관리합니다.
4. **스타일링:** 기존 프로젝트의 스타일에 맞춰 일관성을 유지합니다.
5. **문서화 및 응답:** 모든 프론트엔드 작업 관련 문서와 응답은 한글로 작성합니다.
6. **테스트:** 가능한 경우 Vitest 등을 활용한 컴포넌트 테스트를 우선 작성합니다.
