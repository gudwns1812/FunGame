# 에러 코드 명세 (error.md)

이 문서는 실제 백엔드 `ErrorCode` enum 및 `support/error` 시스템과 동기화된 에러 명세입니다.

## 비즈니스 에러 코드 (ErrorCode)

| 코드 | 설명 | HTTP 상태 코드 | 의미 및 대응 |
| :--- | :--- | :--- | :--- |
| **`G001`** | 방 인원 초과 | 400 Bad Request | 방의 최대 인원(`maxPlayers`)이 찼습니다. |
| **`G002`** | 방을 찾을 수 없음 | 400 Bad Request | 유효하지 않은 `roomId`로의 요청입니다. |
| **`G003`** | 방이 비어있음 | 400 Bad Request | 플레이어가 없는 방에서 게임을 시작할 수 없습니다. |
| **`G004`** | 방장 권한 없음 | 403 Forbidden | 방장이 아닌 유저가 제어 API(시작 등)를 호출했습니다. |
| **`G005`** | 분산 락 획득 실패 | 500 Internal Error | 동시성 처리 중 일시적인 오류가 발생했습니다. |
| **`G006`** | 이미 게임이 진행 중 | 400 Bad Request | 이미 시작된 방에는 입장하거나 다시 시작할 수 없습니다. |
| **`E500`** | 서버 내부 오류 | 500 Internal Error | 예상치 못한 시스템 예외가 발생했습니다. |

### 에러 응답 예시
```json
{
  "result": "FAIL",
  "data": null,
  "error": {
    "code": "G004",
    "message": "방장 권한이 없습니다."
  }
}
```

---
*참고: 모든 비즈니스 예외는 `CoreException`을 통해 발생하며, 전역 예외 처리기에 의해 위 포맷으로 변환됩니다.*
