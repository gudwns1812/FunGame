# 작업 계획서: 준비 상태 동기화 및 방 상태 연동 개선

## 1. 개요
준비(Ready) 버튼 클릭 시 본인의 상태가 화면에 즉각 반영되지 않는 문제를 해결하기 위해, 서버의 방 상태 변경 이벤트를 수신할 때마다 플레이어 리스트와 각 플레이어의 상태를 전체 동기화하는 방식으로 개선합니다.

## 2. 주요 변경 사항

### 2.1. 프론트엔드 (React)
- **상태 업데이트 로직 개선:** 서버로부터 `ROOM_UPDATED` 등 방 정보 변경 메시지를 받을 때, `GameRoom` 객체의 플레이어 리스트를 최신화하여 UI에 반영합니다.
- **준비 상태 렌더링:** `WaitingRoom` 컴포넌트에서 각 플레이어 카드에 표시되는 'READY' 라벨이 서버에서 내려준 `ready` 불리언 값을 정확히 따르도록 보장합니다.
- **즉각적 피드백 (선택적):** 네트워크 지연을 고려하여, 액션 송신 직후 로컬 상태를 먼저 변경(Optimistic Update)할지 검토합니다 (우선은 서버 동기화를 최우선으로 함).

### 2.2. 테스트 가이드라인 (TDD)
- `WaitingRoom.test.tsx`에 '본인이 준비 버튼을 눌렀을 때 상태가 반영되는지'에 대한 테스트 케이스를 추가하거나 보완합니다.

## 3. 작업 절차
1. `plan.md` 생성 및 사용자 리뷰 (현재 단계)
2. `WaitingRoom.test.tsx`에 실패하는 테스트 케이스 작성
3. `WaitingRoom.tsx` 및 관련 훅 (`useGameLogic.ts`) 수정하여 서버 데이터 연동 강화
4. 테스트 통과 확인 및 최종 UI 검증

## 4. 기대 효과
- 본인 및 다른 플레이어의 준비 상태가 항상 최신 서버 데이터와 일치하게 됨.
- 방장 전환, 플레이어 입퇴장 등 다른 방 상태 변경 시에도 일관된 UI 제공 가능.
